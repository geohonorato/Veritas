Guia de Ajuda e Suporte do WorkWise

Visão Geral do Sistema

O WorkWise é um sistema completo de controle de acesso e gestão de presença, utilizando tecnologia biométrica para garantir segurança e precisão. Ele é composto por um dispositivo de hardware inteligente (ESP + Sensor Biométrico) e uma aplicação desktop intuitiva para gerenciamento. Com o WorkWise, você pode facilmente cadastrar novos usuários, monitorar atividades em tempo real e gerenciar as configurações do sistema.

Como Usar o WorkWise

1. Início (Dashboard)

  • Visualize rapidamente as estatísticas de presença (presentes, ausentes, Total de alunos) para o dia atual.
  • Acompanhe as últimas atividades de entrada e saída em tempo real no feed de atividades recentes.
  • Utilize o calendário interativo para selecionar e visualizar atividades em datas específicas.

2. Atividade

  • Acesse a lista completa e detalhada de todas as atividades de entrada e saída registradas pelo sistema.
  • As atividades são organizadas cronologicamente e incluem informações como nome do usuário, turma, tipo de registro (Entrada/Saída) e o horário exato.

3. Usuários

  • Cadastrar Novo Usuário: Clique no ícone para abrir o formulário. Preencha os dados solicitados (Nome, Matrícula, Turma, Email) e siga as instruções no sensor biométrico para registrar a digital do novo usuário.
  • Sincronizar Usuários: Utilize o ícone para importar para o sistema usuários que podem ter sido cadastrados diretamente no dispositivo ou em outro ambiente.
  • Editar Usuário: Clique no ícone (três pontos) ao lado do usuário na tabela. Uma janela pop-up permitirá que você atualize o nome, matrícula, turma e email do usuário.
  • Excluir Usuário: Dentro do modal de edição de usuário, você encontrará um botão "Excluir". Ao clicar nele, o usuário será permanentemente removido do banco de dados do aplicativo e sua digital correspondente será apagada do sensor biométrico.
  • Buscar Usuário: A barra de busca permite localizar rapidamente usuários por nome, matrícula ou ID.

4. Configurações

  • Nesta seção, você pode selecionar e configurar a porta serial correta para estabelecer a comunicação entre a aplicação desktop e o dispositivo ESP.
  • Um indicador de status de conexão informa se a comunicação está ativa e funcionando corretamente.

Solução de Problemas Comuns

"Sensor não responde" ou "Erro de comunicação serial"

  • Verifique se o dispositivo ESP está ligado, conectado fisicamente ao computador e recebendo energia.
  • Na aba "Configurações", certifique-se de que a porta serial correta está selecionada e que não há outros programas utilizando-a.
  • Tente reiniciar o dispositivo ESP e o aplicativo WorkWise.

"Digital não reconhecida" ou "Erro ao processar imagem"

  • Assegure-se de que o dedo (e o sensor) estejam limpos e secos.
  • Pressione o dedo firmemente sobre o sensor, cobrindo completamente a área de leitura, e mantenha-o imóvel por alguns segundos.
  • Tente cadastrar a digital novamente, prestando atenção às instruções e à posição do dedo.

Perguntas Frequentes (FAQ)

Posso usar o WorkWise em vários computadores?

Atualmente, a aplicação desktop do WorkWise é projetada para ser executada localmente em um único computador. O banco de dados de usuários e atividades é armazenado localmente. Para uso em múltiplos locais ou por múltiplos usuários simultaneamente, seria necessária uma solução de banco de dados centralizada e uma arquitetura diferente.

O que acontece se a conexão com o sensor for perdida?

Se a conexão for perdida, o sistema não conseguirá registrar novas atividades biométricas ou gerenciar digitais no sensor. No entanto, os dados de usuários e atividades já armazenados no aplicativo (no arquivo database.json) permanecerão intactos. Reconecte o dispositivo e a porta serial para restaurar a funcionalidade completa.

Como faço para fazer backup dos dados?

O banco de dados do aplicativo é um arquivo simples chamado database.json, localizado na pasta data/ do projeto. Você pode copiá-lo para um local seguro para fazer backup. Para restaurar, basta substituir o arquivo database.json existente pelo seu backup.

Arquitetura do Software (Informações Técnicas)

Firmware (Dispositivo Embarcado)

O cérebro do dispositivo de hardware é um microcontrolador (ESP8266/ESP32) programado em C++ utilizando o framework Arduino através do PlatformIO.

  • Linguagem: C++ (Arduino)
  • Bibliotecas Principais: Adafruit_Fingerprint (para o sensor biométrico), RTClib (para o relógio de tempo real), LittleFS (para armazenamento de dados), e ArduinoJson (para comunicação serial).
  • Funcionalidades: Gerencia o cadastro e a verificação de impressões digitais, armazena dados de usuários e logs de atividade em um sistema de arquivos interno e comunica-se com a aplicação desktop via porta serial usando o formato JSON.

Aplicação Desktop (Painel de Controle)

A interface de gerenciamento é uma aplicação desktop multiplataforma construída com Electron.

  • Tecnologias: Electron, Node.js, HTML5, Tailwind CSS, JavaScript.
  • Funcionalidades: Permite listar e cadastrar novos usuários, visualizar a tabela de atividades, e configurar a porta de comunicação com o hardware. A comunicação é feita através da biblioteca `serialport`.

Montagem do Hardware

Os componentes de hardware devem ser conectados ao microcontrolador (assumindo um NodeMCU/ESP8266) da seguinte forma:

  • Sensor Biométrico (R307):
    • VCC: 5V
    • GND: GND
    • TX (Sensor): Pino D7 (GPIO 13) do ESP
    • RX (Sensor): Pino D6 (GPIO 12) do ESP
  • Módulo de Relógio (RTC DS3231):
    • VCC: 3.3V
    • GND: GND
    • SDA: Pino D2 (GPIO 4) do ESP
    • SCL: Pino D1 (GPIO 5) do ESP
